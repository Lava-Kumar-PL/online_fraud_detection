Fraud Detection System

The surge in online credit/debit card transactions has led to an upsurge in fraud. To combat this, we're employing classification algorithms like Random Forest. Using a vast dataset, we'll train and evaluate these models, selecting the one with the highest accuracy, precision, and a low false positive rate. The chosen model will be saved in pkl format. We'll integrate it into a Flask web application for real-time detection. This project aims to enhance fraud detection, save costs, and offer real-time protection.

Transaction Data Ranges Based on EDA

Fraudulent Transactions:

  • step: Typically between 1 and 743
  • type: More likely to be 2, 3, or 4
  • amount: Higher amounts, typically above 1,467,967.30
  • Sendersoldbalance: Lower balances, typically below 1,649,667.61
  • Reciversoldbalance: Can vary, but often lower balances, typically below 544,249.62

Non-Fraudulent Transactions:

  • step: Typically between 1 and 718
  • type: More likely to be 0 or 1
  • amount: Lower amounts, typically below 178,197.04
  • Sendersoldbalance: Higher balances, typically above 832,828.71
  • Reciversoldbalance: Can vary, but often higher balances, typically above 1,101,420.87
